Why GitHub Pages
Claude can only read files that are publicly accessible on the web. Local files are invisible.
Archive Structure

The archive is organized in three levels. Claude files each entry automatically — you never need to specify where it goes beyond the date.

Quarter → Date → Source Card Label → Pill entry

The source card label (e.g. This and That, On Our Radar) is detected automatically from the portal HTML — you don't need to supply it.

① Session Archive
Moves current agenda into the sessions grid
Step 1 — Archive the session immediately after it ends
Archive Session

Claude reads the session date and agenda topics from the live portal and moves them into the correct seasonal grid. The three pills are created with no links — placeholders until the URLs are ready.

🎥 Recording 📊 Summary 📄 Afterwords

Pills appear immediately — links added in Step 2

Step 1 variant — Archive with trending link
Archive Session | Trending: [url]

If the Trending report is already published, include it now. The Recording, Summary, and Afterwords pills are still created as placeholders.

📈 Trending 🎥 Recording 📊 Summary 📄 Afterwords
Step 2 — Fill in the links when ready
Update Session Links: Recording: [url] | Summary: [url] | Afterwords: [url]

Claude finds the session entry with empty pills — there is only ever one — and fills them in. Supply only the links that are ready; the rest remain as placeholders.

🎥 Recording 📊 Summary 📄 Afterwords
Optional — Archive and clear agenda in one step
Archive Session and clear agenda

Archives the session and empties the current agenda card in the same operation, leaving it ready for the next session's topics.

What Claude reads automatically from the portal

Session date → Agenda topics & links → Correct seasonal grid → Correct column

The seasonal section (Spring / Winter / Autumn) and column placement are determined from the session date — no need to specify them.

② For Your Consideration Archive
Moves resource pills into FYC_Archive.html
Archive a single pill
Archive Pill/Reference: [date] | [pill text]

Example: Archive Pill/Reference: May 2026 | State of Play of the Administration
Claude fetches the portal, finds the pill by text, extracts its URL and source card label, then inserts the entry into the archive.

Archive from a screenshot — recommended
Archive Pill/Reference: [date]
Screenshot method Upload a screenshot showing any number of pills alongside this command. Claude reads all pill texts from the image and archives every one in a single pass — no need to type them out.
Archive and remove from portal
Archive Pill/Reference: [date] and remove from portal

Works with a single pill text or a screenshot. Claude returns both files updated — the archive with the new entries, the portal with the pills removed. Upload both to GitHub.

Remove only — no archive
Remove from Portal: [pill text]

Removes the pill from the live portal without adding it to the archive. Returns updated portal HTML only.

What Claude Does Automatically

Session Archive

1Fetches the live portal from GitHub Pages
2Reads the current session date and agenda topics
3Determines the correct seasonal section and column from the date
4Creates session entry with date badge, topics, and three link-free placeholder pills
5On Update Session Links, finds the entry with empty pill hrefs and fills them in

FYC Archive

1Fetches both live portal and FYC_Archive.html
2Finds the pill by text and extracts its href
3Records the source card label as provenance
4Files under correct quarter, date block, and card group
5Creates new quarter or date block if needed
